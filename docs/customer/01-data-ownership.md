## あなたのデータと所有権

> **対象読者**: MeX Next を使うお客様
> **読了時間**: 約 4 分

MeX Next の運用データは、あなたの GitHub にある専用 private repo に保存されます。運営者は作業のために collaborator として入りますが、データの持ち主はあなたです。

## 1. 何が保存されるか

専用 repo には次のような情報が入ります。

| 種類           | 例                                   |
| -------------- | ------------------------------------ |
| アカウント設定 | 投稿ペース、口調、追跡対象           |
| 運用状態       | 予約中の投稿、今日スキップするか     |
| 投稿データ     | 下書き、公開済み本文、簡単な反応記録 |
| bot 用メモ     | あなたのブランド、話し方、方針       |

X / Discord / GitHub の token などの秘密情報は repo には入れません。秘密情報は Doppler という secret 管理サービスに置きます。

## 2. いつでも削除できます

あなたは GitHub repo をいつでも削除できます。

```bash
gh repo delete <あなたのGitHub>/<account>-x-ops
```

削除すると、bot の運用データも完全に消えます。運営者は削除を止めません。削除後は復元できないため、必要なら先に export してください。

削除したい時は、運営者に「MeX の repo を削除したい」と伝えてください。bot 停止、VPS 上の clone 削除、Doppler secret 削除まで行います。

## 3. いつでも持ち出せます

データを手元に持ち出すには、repo を clone するだけです。

```bash
gh repo clone <あなたのGitHub>/<account>-x-ops
```

別の運営者や別ツールへ移したい場合も、この repo があれば主要データを移せます。秘密情報は安全のため、移管時に再発行するのが基本です。

## 4. 運営者側に残るもの

運営者側には、bot を動かすための VPS 設定、Doppler の service token、短期間のログが残ります。投稿本文やアカウント状態の別バックアップを運営者が持ち続ける運用にはしません。

調査のために一時的なコピーが必要な場合は、目的が終わったら削除します。

## 5. bot 外へ持ち出したい時

「投稿データを分析したい」「別ツールに移したい」「契約終了前に全部ほしい」と伝えてください。運営者は専用 repo の export 手順を案内します。

あなたが GitHub に慣れていない場合は、運営者が archive を作って渡せます。その場合も、渡した後に運営者側の一時ファイルは削除します。
