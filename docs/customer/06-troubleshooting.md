## よくあるトラブル

> **対象読者**: 全顧客
> **前提**: なし (困った時に開く)
> **読了時間**: 必要なところだけ拾い読み

困った時はこのページを開いてください。解決しなければ Discord で運営にメンションするのが最短です。

## 1. bot が反応しない

### 原因 1: bot がオフライン

bot のステータスを Discord で確認してください。緑のドットが付いていなければ bot プロセスが落ちています。

```text
対応: 「bot が落ちています」と運営にメンション
```

### 原因 2: メンションがついていない

channel で書く場合は **`@mex-...` メンション必須**。DM なら不要です。

### 原因 3: `/` で始まっている

`/` で始まるメッセージは slash command として扱われ、自然言語ルートを通りません。`/` を外して書き直してください。

### 原因 4: thread の中

進行中の onboarding / 投稿案編集 / ターゲット設定 thread の中では、専用フローが処理を奪います。意図した動作と違う場合は `cancel` で抜けてください。

## 2. 「うまく聞き取れませんでした」

LLM が intent 抽出に失敗しました。次のどれかが該当します。

| 状況 | 対策 |
| --- | --- |
| 文が短すぎ ("おはよう" 等) | 操作意図を含めて書き直す |
| 文が長すぎ / 複数操作が混じる | 1 操作ずつに分ける |
| 表現が独自すぎ | 例文に近い言い回しに変える |
| LLM タイムアウト | 30 秒待って再送 |

`/mex help` で操作一覧を見るか、[01-talk-to-bot.md](./01-talk-to-bot.md) の例文を参考に書き直します。

## 3. 朝の draft が来ない

### 原因 1: skip_today

「今日いらない」を当日中に押した場合、その日の draft は届きません。`状態確認` で `skip_today: true` になっていないか確認。

### 原因 2: cadence=light で同日 publish 済

light は 1 日 1 本。すでに publish 済みならその日の追加 draft は出ません。`新しい投稿作って` で手動生成は可能。

### 原因 3: 5-axis 品質判定が連続失敗

repair 2 回失敗で `awaiting_decision` まで進むので、何かしらの通知は出るはずです。完全な無音は次の原因が考えられます。

### 原因 4: 内部エラー

bot が `failed_terminal` で止まっている可能性。運営にメンションしてください。

## 4. 予約時刻が想定と違う

### hot_zones の random offset

bot は hot_zones 内で **±30 分の random offset** をかけて時刻を選びます。06:00-09:00 hot zone なら 06:30 になることも 08:45 になることもあります。これは仕様です。

### 衝突回避

±30 分以内に既存予約があると再抽選されます。5 回失敗で翌日 fallback します。

具体的にどの時刻にしたい時は、現状の slash command でも一発指定はできません。`時刻指定の予約はサポートされていない` ので、cadence の hot_zones を狭めることで対処します (運営に依頼)。

## 5. 投稿が予約時刻に出ない

### 原因 1: X API rate limit

X API の rate limit に当たって publish が遅れることがあります。`mex-publish-<account>.timer` の failsafe (5 分間隔再試行) で 5-15 分後には出るはずです。

### 原因 2: tweet が long すぎ (280 over)

5-axis judge の length_fit で弾かれ awaiting_decision になっているはず。本文を確認してください。

### 原因 3: bot が落ちている

Discord で bot のステータス確認 + 運営にメンション。

## 6. リプ通知が来ない

### 原因 1: poll 間隔

X API poll は 30 分間隔。最大 30 分の遅延があります。

### 原因 2: risk_classify で medium / high

medium / high は operator にだけ通知され、あなたには summary か notice しか出ません。これは仕様です。

### 原因 3: rate limit

X API の rate limit 中は poll が一時停止します。1-2 時間後に再開。

## 7. 通知が多すぎる

```text
あなた: 自動運用一括 OFF にして
bot:    automation 一括 OFF にしますか？
        [はい] [いいえ]
```

(ただし一括 OFF は実装予定の機能です。現状は cadence を light に下げる + skip_today で対応)

## 8. 過去の投稿を取り消したい

X API では publish 済みのツイートを bot が削除する機能は提供していません (X側の仕様)。X 公式アプリから手動削除してください。`state.json` 上は publish 履歴が残るため、dedup に影響します (削除済みでも prefix が判定対象)。

## 9. account を移行したい

別 VPS への移行 / 別 Discord サーバーへの移行は **operator 作業** です。運営にメンションして移行依頼を出してください。

## 10. 完全に停止したい

```text
あなた: しばらく止めて
bot:    停止期間を教えてください (例: 1 週間 / 12/31 まで)
```

長期停止は operator が systemctl で stop してくれます。運営にメンションしてください。

## 11. 連絡先

それでも解決しない時は、Discord で運営アカウントにメンション。返答時間は **平日 09:00-18:00 JST** が目安です。
